#### Calendar System: Imperial Meridian Calendar

The Imperial Meridian Calendar is the official timekeeping system of the Meridian Empire. The calendar is organized hierarchically with the following time divisions:

**Epochs** are the largest division, marking major historical eras. The current epoch is the Third Age of Meridian.

**Cycles** are the equivalent of years in the Imperial calendar. Each Epoch contains many Cycles. Currently we are in Cycle 1847 of the Third Age. One Cycle equals 360 rotations.

**Seasons** divide each Cycle into quarters. There are 4 Seasons per Cycle: Spring, Summer, Autumn, and Winter. Each Season contains 90 rotations.

**Rotations** are the equivalent of days. There are 360 Rotations per Cycle, with 90 Rotations per Season.

**Segments** are the hours of each Rotation. There are 24 Segments per Rotation, numbered from 0 to 23.

The current date is Cycle 1847, Autumn Season, Rotation 45, Segment 12.

## Post-Analysis Setup Steps

After running content analysis and accepting the suggestions:

### Step 1: Verify Entity Creation
Check that the following were created:
- ✅ 1 Calendar card: "Imperial Meridian Calendar"
- ✅ 3 Timeline cards: "Commander Vex's Military Campaign", "Merchant Kael's Trade Route", "Scientist Lyra's Research Expedition"
- ✅ 21 Scene cards (7 per timeline)
- ✅ Relationships: Each scene → its timeline

### Step 2: Configure Calendar System
1. Open the "Imperial Meridian Calendar" card
2. In Details tab, verify CalendarSystem was created with proper time divisions
3. If not auto-created, manually enter the hierarchy:
   - Epoch (no parent)
   - Cycle (parent: Epoch, count: varies)
   - Season (parent: Cycle, count: 4)
   - Rotation (parent: Season, count: 90)
   - Segment (parent: Rotation, count: 24)

### Step 3: Link Timelines to Calendar
1. Open each Timeline card
2. Flip to back (ellipsis menu)
3. Select "Imperial Meridian Calendar" from the calendar picker
4. Set epoch date and description (specified in the prompt for each timeline)

### Step 4: Set Scene Temporal Positions
**Note:** This step currently requires manual entry via the Timeline view.

For each Timeline:
1. Open the Timeline card
2. Go to Timeline tab (should show temporal mode after calendar assignment)
3. Click "Edit Positions..." menu
4. For each scene, set temporal position and duration as specified in the prompt

**Temporal positions and durations are clearly specified in the prompt text** - look for phrases like:
- "takes place on Cycle 1847, Spring Season, Rotation 1, Segment 8"
- "lasts for 4 segments"

### Step 5: View Multi-Timeline Graph
1. Open the "Imperial Meridian Calendar" card
2. Navigate to Tab 3 (Timeline)
3. You should see the Multi-Timeline Graph with all three timelines
4. Scenes from all timelines on the shared calendar axis
5. Notable overlaps to verify:
   - Commander Vex and Kael both at Outpost Seven (Summer, Rotation 44-45)
   - Lyra evacuates when Vex's combat approaches (Summer, Rotation 15)
   - Kael delivers to Lyra's expedition (Autumn, Rotation 35-36)
   - Lyra warns Vex about threat (Autumn, Rotation 60)

## Troubleshooting

**If Calendar System isn't extracted:**
- The calendar extraction might need more explicit structure
- Manually create the time divisions in the Calendar card Details tab

**If Scenes don't link to Timelines:**
- The relationship language might not match inference patterns
- Manually create relationships: Scene → Timeline using "describes/described-by"

**If temporal positions aren't set:**
- This is expected - temporal position setting currently requires manual entry
- Follow Step 4 above to set positions via Timeline view

**If Characters are created instead of Timelines:**
- This is possible depending on entity extraction
- Convert card kind to "Timelines" after creation

## Why This Works

This prompt is designed to work with Cumberland's existing AI features:

1. **Entity Extraction** (Phase 5) - Recognizes named entities in the text
   - Timelines described as narrative arcs
   - Scenes explicitly named with "is a scene"
   - Calendar system described with structure

2. **Relationship Inference** (Phase 6) - Parses relationship language
   - "appears in", "occurs in", "is a scene in" → triggers scene→timeline relationships
   - Patterns match the 13 relationship detection patterns

3. **Calendar Extraction** (Phase 7) - Extracts calendar structure
   - Hierarchical time division descriptions
   - Clear parent-child relationships
   - Count specifications ("4 Seasons per Cycle")

The prompt is written as a worldbuilding document, which is exactly what Cumberland's content analysis is designed to process!
