# Unified Fix: DR-0001, DR-0002, DR-0003

**Date:** 2025-12-04
**Status:** Analysis Complete - Ready for Implementation

## Root Cause Analysis

All three DRs stem from the same root cause: **SwiftUI is not detecting changes to `PKTool` properties**.

### The Problem

In `DrawingCanvasView.swift:51-57`, the `PencilKitCanvasView` is created like this:

```swift
PencilKitCanvasView(
    drawing: $canvasState.drawing,
    tool: canvasState.selectedTool,  // ⚠️ PKTool is not Equatable!
    isRulerActive: canvasState.isRulerActive,
    canvasSize: canvasState.canvasSize,
    zoomScale: canvasState.zoomScale
)
```

**Issue:** `PKTool` does not conform to `Equatable`, so when:
- Tool type changes (pen → pencil → marker)
- Tool color changes
- Tool width changes

SwiftUI doesn't realize the `tool` property has changed and doesn't call `updateUIView()` to update the PKCanvasView's tool.

### Why Eraser Works

The eraser works because selecting it changes `selectedToolType` to `.eraser`, and the `updateTool()` method creates a completely different tool type (`PKEraserTool` instead of `PKInkingTool`), which SwiftUI does detect as a change.

### Evidence from DRs

- **DR-0001:** Black appears white because the tool isn't updating with the new color
- **DR-0002:** Tool type selection doesn't work because PKInkingTool changes aren't detected
- **DR-0003:** Width changes don't work because the tool isn't being updated

## Solution

Add a change tracker to force SwiftUI to update the PencilKit canvas when tool properties change.

### Option A: Add .id() Modifier (Recommended)

Force view recreation when tool changes by adding a tool change counter:

**Step 1:** Add counter to `DrawingCanvasModel` (line ~140):

```swift
/// Tool change counter to force SwiftUI updates
var toolChangeCounter: Int = 0
```

**Step 2:** Increment counter in `updateTool()` method (line ~233):

```swift
func updateTool() {
    #if canImport(PencilKit)
    let pkColor: PKInkingTool.InkType

    switch selectedToolType {
    case .pen:
        pkColor = .pen
    case .pencil:
        pkColor = .pencil
    case .marker:
        pkColor = .marker
    case .eraser:
        selectedTool = PKEraserTool(.vector)
        toolChangeCounter += 1  // ✅ Add this
        return
    case .lasso:
        selectedTool = PKLassoTool()
        toolChangeCounter += 1  // ✅ Add this
        return
    }

    #if canImport(UIKit)
    selectedTool = PKInkingTool(pkColor, color: selectedColor.toPencilKitColor(), width: selectedLineWidth)
    #elseif canImport(AppKit)
    selectedTool = PKInkingTool(pkColor, color: NSColor(selectedColor), width: selectedLineWidth)
    #endif
    toolChangeCounter += 1  // ✅ Add this
    #endif
}
```

**Step 3:** Add `.id()` modifier to `PencilKitCanvasView` (line ~51):

```swift
PencilKitCanvasView(
    drawing: $canvasState.drawing,
    tool: canvasState.selectedTool,
    isRulerActive: canvasState.isRulerActive,
    canvasSize: canvasState.canvasSize,
    zoomScale: canvasState.zoomScale
)
.id(canvasState.toolChangeCounter)  // ✅ Add this
```

### Option B: Use onChange Modifier (Alternative)

Add explicit change detection for tool properties:

```swift
PencilKitCanvasView(...)
.onChange(of: canvasState.selectedToolType) { _, _ in
    // Force update
}
.onChange(of: canvasState.selectedColor) { _, _ in
    // Force update
}
.onChange(of: canvasState.selectedLineWidth) { _, _ in
    // Force update
}
```

**Disadvantage:** More verbose and requires tracking each property individually.
